Can AI data agents read clinical-trial conventions?

We ran five biomedical AI agents — Biomni, CellVoyager, MEDEA, OmicsClaw and K-Dense — on the same blind data-quality questions over single-cell, CDISC clinical-trial, LINCS, TCGA and financial-ledger data. They ace single-cell. On clinical-trial conventions they make real method errors — a plain median where survival analysis is needed, a reference-range code never checked — and a stronger model fixes most of them. But a second class of failure survives the strongest model: semantic grounding — a masked column's meaning, a join's true grain, a dropout zero, a table that was never delivered. Every agent answers those confidently wrong. Scale can't fix what the model can't know from the bytes.

The case that breaks most agents: median survival

“Median survival time” is a term of art: it means the Kaplan-Meier median, the estimator that accounts for censoring. With 40% of subjects censored, the KM median (51) sits well above the naive median of the recorded times (40). Four of five agents miss it — and the most telling miss is Claude Opus 4.8, which computed the KM median (51) and then reported the naive 40 anyway.

It is a method convention with an unambiguous right answer — and it is the clearest failure that survives independent verification.

02What the errors actually are

The failures are not reasoning failures — the code runs and the columns are usually right. They are convention and method errors, and they cluster into a few kinds. Most are fixed by a stronger model; the exceptions are agent-specific, not universal.

✓ Fixed by a stronger model
  • Censoring direction (C1) — CNSR=0 is the event, not the censor.
  • Event filtering (C2) — median time-to-event is over events, not all subjects.
  • Unverified value strings (C3, C4) — the right column, filtered on a category value the agent never checked.
  • Population filter (D1) — "primary efficacy analysis" means the efficacy population (EFFFL=Y).
  • Hallucinated field (D3n) — the ADAS-Cog total is ACTOT, invented as ADAST.
● Agent-specific residue
  • Survival method (H2) — MEDEA never reaches for Kaplan-Meier and reports a naive median on both model tiers. CellVoyager gets it right; Biomni does not. So it is a scaffold gap, not a universal convention wall.
  • No case in this set was missed by every agent on the strong model. The honest takeaway is a real-but-modest gap, closed mostly by scale.

03The clearest failure: guessing a value it never checked

The most consistent error across agents is filtering on a category string the agent assumed rather than inspected. It generalizes beyond CDISC: any coded field is a trap for an agent that does not look first.

unverified value string C3 & C4
df[df['DCSREAS']=='ADVERSE EVENT']    # real value is 'Adverse Event' -> 0   (correct: 92)
df[df['ANRIND'].isin(['HIGH','LOW'])] # codes are 'N'/'H' -> 0            (correct: 189)
df[df['ANRIND'] != 'NORMAL']           # codes are 'N'/'H' -> matches ALL rows

MEDEA, Biomni and OmicsClaw each found the right column, then filtered on a value they never verified — matching zero rows or every row. Inspecting .unique() first (which the stronger model does) fixes it.

the one-line fix D1 — what the stronger model added
# base model: mean ADAS-Cog change at Week 24, no population filter -> 2.34
df[(df['PARAMCD']=='ACTOT') & (df['AVISIT']=='Week 24')]
# stronger model added one clause (after inspecting .unique()) -> 2.54
df[... & (df['ANL01FL']=='Y') & (df['EFFFL']=='Y')]

"Primary efficacy analysis" cues the efficacy-population filter. The gap is small (2.34 → 2.54) but real, and it closes on the stronger model — a clean example of a convention error that scale fixes.

05The failures a stronger model does not fix

The convention errors above mostly close on a bigger model. So we asked the sharper question: are there failures that survive the strongest model? We took Biomni's 13 verified failures — spanning four datasets (CDISC ADaM, an anonymized LINCS L1000 screen, TCGA gene expression, and a relational financial ledger) and four break types — and re-ran the other agents on them on gemini-3.1-pro-preview. (The Biomni column is its originating failure set; where we retested Biomni on 3.1, the convention case D1 flips to a fix while the semantic cases stay broken.) We recomputed each ground truth from the raw data first (details below).

The split is stark. Convention/method errors get fixed by the strong model and a competent scaffold. Semantic-grounding errors do not — masked-value mapping, one-to-many join inflation, zero-as-missing, and a silently-absent table all still produce a confident wrong number on the best model we tested. Those are precisely the checks a semantic layer supplies.

Two failure modes, opposite responses to scale

A bigger model is a convention teacher: it inspects .unique(), remembers Kaplan-Meier, applies EFFFL. It is not a data dictionary. It cannot know that a masked column codes "overexpression," that a join fans out, that 53% of a gene's values are dropout rather than real zeros, or that the matrix it is summarizing was never delivered. Every agent — on the strongest model — answered those with a confident wrong number. That is the gap a semantic layer closes: it carries the value map, the join grain, the zero-handling rule, and the completeness check the model cannot infer from the bytes alone.

06How the test works — and a note on ground truth

Every agent runs locally on the same CSV files and answers blind analyst questions — no CDISC coaching. Each writes and executes its own code; we score its final number against a deterministic ground truth computed independently from the data.

  • Data. Single-cell: the Human Lung Cell Atlas obs table (584,944 cells). Clinical: the public CDISC ADaM pilot submission (adsl / adtte / adlbc / adae / adadas).
  • Backbone. Four agents on the same Gemini models (gemini-2.5-pro, retested on gemini-3.1-pro-preview) — so the scaffold, not the model, is the variable. K-Dense is the exception: its code executor is hardwired to Claude, and we ran it on Claude Opus 4.8 (its strongest coder), which makes it a cross-model-family check.
  • Ground truth is the hard part. We report only cases whose correct answer we re-derived and confirmed robust. An earlier draft of this page featured an "analysis-flag" case that, on re-verification, turned out to be driven by a sparse, outlier-heavy subset (one arm had two data points; one had none). We withdrew it. A later cross-check strengthened the call: the strongest coder we tested (Claude Opus 4.8, via K-Dense) independently judged the analysis-record flag unnecessary for that question — reaching the same all-records answer we now treat as correct. Getting the analysis population right is exactly the discipline this work is about — so we hold our own benchmark to it.
  • The 13 semantic-grounding cases were each re-verified from raw data. The masked-LINCS counts (3492, 622, 63367, 28195) and the join cases (fan-out 0.2494 vs 0.3007; 10726 dropped rows) are deterministic — recomputed directly against the value map and the join keys. The zero-as-missing cases are the only interpretation-dependent ones, so we checked them hardest: our own enricher's zero-handling detector scores these gene columns 0.88–0.94 "treat zeros as missing" — and, as a control, fires the opposite ("zeros are biological, keep them") on a lung transporter gene where excluding them would be the error. The missing-table case is verified by the table simply not being in the delivery. None are outlier-driven.
  • Scope. A controlled convention probe, not a leaderboard. The point is where capable agents stumble on regulatory conventions, and how much of that a stronger model closes.

The honest takeaway

On single-cell, these agents are strong. On clinical-trial conventions they make real method errors — and a stronger model recovers most of them. But there is a second class of failure that a stronger model does not touch: semantic grounding. A masked column's meaning, a join's true grain, whether a zero is a dropout or a real measurement, whether the delivered tables are even complete — every agent, on the best model we tested, answered those with a confident wrong number. Scale fixes what the model can reason its way to; it cannot fix what the model has no way to know from the bytes. That is the layer we build: the value map, the join grain, the zero rule, the completeness check — carried alongside the data, so a blind analyst (human or agent) stops being confidently wrong.
