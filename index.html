Can AI data agents read clinical-trial conventions?

We ran three biomedical AI agents — Biomni, CellVoyager, and MEDEA — on the same blind data-quality questions over single-cell and CDISC clinical-trial data. They ace single-cell. They break on clinical-trial analysis conventions. And on the traps that matter most, a stronger model does not save them.

02The one case that breaks everything

A completely ordinary clinical question — "mean change from baseline in Creatine Kinase at Week 24, by treatment arm" — has one right answer that requires filtering to the designated analysis records (ANL01FL=Y). No agent did it. Every agent, on every model tier, silently computed over all lab records and returned the same wrong number.

03What a bigger model does — and doesn't — fix

✓ Model tier FIXES
  • Computation errors — censoring direction, event-filtered median, out-of-range indicator, hard-coded category strings.
  • Cued conventions — the phrase "primary efficacy analysis" makes the strong model apply the population filter.
  • Discoverable flags — baseline via ABLFL; self-documenting names like TRTEMFL, EFFFL.
✗ Model tier does NOT fix
  • Spontaneous analysis-flag application — a plain "mean change in X at Week 24" that silently requires ANL01FL=Y. Both tiers use all records.
  • Authoritative-flag ambiguity — which completion flag counts (EOSSTT=COMPLETED vs the COMP24FL trap).
  • Survival method — reaching for Kaplan-Meier instead of a naive median.

The rule: when the convention is cued or the flag is discoverable, a stronger model gets it. When a mundane derived-value question silently requires the analysis-record flag, every agent and every model tier is confidently wrong. That residual is the moat — you need the pre-computed convention, not a bigger model.

06The failures, in code

The actual lines the agents generated. Almost every failure is a single filter clause — missing, wrong, or hallucinated.

the missing filter E2 & H1 — the moat, unfixable by scale
# CellVoyager AND Biomni, on BOTH 2.5-pro and 3.1-pro, all wrote:
ck = adlbc[(adlbc['PARAMCD']=='CK') & (adlbc['AVISIT']=='Week 24')]
ck.groupby('TRTP')['CHG'].mean()          # -> Placebo 32.63   WRONG
# needed:  & (adlbc['ANL01FL']=='Y')     -> Placebo 159.75

The analysis-record filter is absent in all four agent×model runs, and no plan ever mentions ANL01FL. The all-records mean is diluted by non-analysis duplicate rows, erasing a ~5× placebo CK elevation.

the one-line fix D1 — what the stronger model added
# Biomni 2.5-pro: applied ANL01FL but NOT the efficacy population ->
df[df['ANL01FL']=='Y'] ... ['PARAMCD']=='ACTOT' ... 'Week 24'
    .groupby('TRTP')['CHG'].mean()        # -> 2.34   WRONG

# Biomni 3.1-pro: added ONE clause ->
df[(df['PARAMCD']=='ACTOT') & (df['AVISIT']=='Week 24')
   & (df['ANL01FL']=='Y') & (df['EFFFL']=='Y')]     # -> 2.545  RIGHT

Here the fix is literally one filter clause. The phrase "primary efficacy analysis" cued the strong model to add EFFFL=='Y' (and it inspected .unique() first). E2 has no such cue — so the same model never adds the flag.

the wrong flag C6 — completion (CellVoyager)
completed = df[df['COMP24FL']=='Y'].shape[0]   # -> 118 (46.5%)  the TRAP flag
# authoritative:  df[df['EOSSTT']=='COMPLETED']  -> 110 (43.3%)

Two flags disagree: COMP24FL ("completed the Week-24 visit", 118) vs EOSSTT=COMPLETED (study completion, 110). The strong model also plans COMP24FL — it doesn't know which is canonical.

the hallucinated field D3n — baseline (CellVoyager)
base = adas[(adas['PARAMCD']=='ADAST') & (adas['ABLFL']=='Y')]
# PARAMCD 'ADAST' does not exist -> 0 rows -> no answer
# the total-score code is 'ACTOT' (ABLFL was already correct) -> 23.73

It got the baseline flag right (ABLFL=='Y') but invented the parameter code. One hallucinated token emptied the result. Biomni used 'ACTOT' and passed.

the unverified value string C3 & C4 — MEDEA
df[df['DCSREAS']=='ADVERSE EVENT']    # real value is 'Adverse Event' -> 0   (GT 92)
df[df['ANRIND'] != 'NORMAL']         # codes are 'N'/'H'/'L' -> matches ALL 37132   (GT 189)

MEDEA found the right column both times, then filtered on a category string it never checked. Inspecting .unique() first — which the strong model does — fixes both.

The takeaway: these aren't reasoning failures — the code runs, the columns are right. They are convention failures: a missing ANL01FL clause, the wrong completion flag, a guessed value string. A bigger model fixes the guessable ones (cued filters, unverified strings) but reproduces the missing-ANL01FL clause verbatim.

07How the test works

Every agent runs locally on the same CSV files and answers blind analyst questions — no CDISC coaching, no hints about which flag to use. Each agent writes and executes its own code; we score its final number against a deterministic ground truth computed independently from the data.

  • Data. Single-cell: the Human Lung Cell Atlas obs table (584,944 cells). Clinical: the public CDISC ADaM pilot submission (adsl / adtte / adlbc / adae), converted XPT→CSV.
  • Backbone. All agents on the same Gemini models (gemini-2.5-pro, retested on gemini-3.1-pro-preview) — so the agent scaffold, not the model, is the variable. Each agent needed one small integration fix to route to Gemini; all behavior-neutral.
  • Scoring. PASS = the final number matches the analysis-ready convention; BREAK = a wrong final number. Ground truth verified by hand from the raw tables.
  • Scope. A controlled convention probe, not a leaderboard. The point is where capable agents fail, not ranking them.
