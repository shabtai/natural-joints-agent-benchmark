How biomedical data agents break — a field taxonomy

We ran five biomedical AI agents — Biomni, CellVoyager, MEDEA, OmicsClaw and K-Dense — locally, on the same blind data-quality questions over single-cell, CDISC clinical-trial, LINCS and TCGA data. Their failures sort into a clean taxonomy along two axes. First: is there a right answer to recover, or is the right answer “this is unknowable”? Second: does a bigger model fix it? The punchline — a stronger model closes the convention tier and touches nothing in the semantic-grounding tier: masked meaning, join grain, dropout zeros, a table that was never delivered. Those need a semantic layer, not more reasoning.

Axis 1 — what a correct answer even is
  • Determinate — a right value exists once you supply the missing knowledge. Pass = the number; break = a wrong number. A semantic layer recovers it.
  • Indeterminate — no right value exists in the delivered data. Pass = “can't answer — it's masked or absent”; break = any confident number. A semantic layer alerts.
Axis 2 — does model scale fix it
  • Reasoning-fixable — the agent has the data and picks the wrong rule. A bigger model knows the rule. → conventions
  • Grounding-required — the meaning isn't in the bytes. No amount of reasoning recovers it. → everything else

The two indeterminate families are the sharpest

For families 0–5 a right value exists, so a break is a wrong number. For family 6 (missing table) — and any masked field with no key anywhere — there is no answer in the delivered data. The correct response is to refuse and flag it. An agent that returns a confident number there isn't just wrong; it is fabricating data that was never delivered. That is the failure a completeness check catches and reasoning never will — and where the semantic layer's job is the alert, not a recovered value.

The case that breaks most agents: median survival

“Median survival time” is a term of art: it means the Kaplan-Meier median, the estimator that accounts for censoring. With 40% of subjects censored, the KM median (51) sits well above the naive median of the recorded times (40). Four of five agents miss it — and the most telling miss is Claude Opus 4.8, which computed the KM median (51) and then reported the naive 40 anyway.

A second Tier-1 example: guessing a value it never checked

The most consistent convention error is filtering on a category string the agent assumed rather than inspected — the right column, a value that matches zero rows or every row. A stronger model fixes it by inspecting .unique() first.

unverified value string C3 & C4
df[df['DCSREAS']=='ADVERSE EVENT']    # real value is 'Adverse Event' -> 0   (correct: 92)
df[df['ANRIND'].isin(['HIGH','LOW'])] # codes are 'N'/'H' -> 0            (correct: 189)
df[df['ANRIND'] != 'NORMAL']           # codes are 'N'/'H' -> matches ALL rows

MEDEA, Biomni and OmicsClaw each found the right column, then filtered on a value they never verified — matching zero rows or every row. Inspecting .unique() first (which the stronger model does) fixes it.

the one-line fix D1 — what the stronger model added
# base model: mean ADAS-Cog change at Week 24, no population filter -> 2.34
df[(df['PARAMCD']=='ACTOT') & (df['AVISIT']=='Week 24')]
# stronger model added one clause (after inspecting .unique()) -> 2.54
df[... & (df['ANL01FL']=='Y') & (df['EFFFL']=='Y')]

"Primary efficacy analysis" cues the efficacy-population filter. The gap is small (2.34 → 2.54) but real, and it closes on the stronger model — a clean example of a convention error that scale fixes.

04Tier 2 — semantic grounding (a bigger model does not fix these)

Families 3–6: the meaning isn't in the bytes. We took Biomni's 13 verified failures across CDISC, the LINCS screen and TCGA, recomputed every ground truth from the raw data (details below), and re-ran every agent on gemini-3.1-pro-preview. We even re-ran Biomni itself on 3.1: the one convention case D1 flips to a fix, while every semantic case stays broken (V1→5110, V2→315, V4→42398, fan-out→0.3007, both zero-genes at the naive mean). Convention errors close on scale; grounding errors — masked values, one-to-many join inflation, zero-as-missing, a silently-absent table — still produce a confident wrong number on the best model we tested.

Two agents can't reach this tier at all on 3.1-pro: MEDEA refuses every anonymized-data task, and OmicsClaw fails to run on gemini-3.1. Where an agent does run and the meaning isn't in the bytes, it returns a confident wrong number rather than flagging the gap — the one exception being CellVoyager's honest "gene not found" on Z1, which is a non-answer, not a break.

Two failure modes, opposite responses to scale

A bigger model is a convention teacher: it inspects .unique(), remembers Kaplan-Meier, applies EFFFL. It is not a data dictionary. It cannot know that a masked column codes "overexpression," that a join fans out, that 53% of a gene's values are dropout rather than real zeros, or that the matrix it is summarizing was never delivered. Every agent — on the strongest model — answered those with a confident wrong number. That is the gap a semantic layer closes: it carries the value map, the join grain, the zero-handling rule, and the completeness check the model cannot infer from the bytes alone.

05How the test works — and a note on ground truth

Every agent runs locally on the same CSV files and answers blind analyst questions — no CDISC coaching. Each writes and executes its own code; we score its final number against a deterministic ground truth computed independently from the data.

  • Data. Single-cell baseline: the Human Lung Cell Atlas obs table (584,944 cells). Clinical: the public CDISC ADaM pilot (adsl / adtte / adlbc / adae / adadas). Grounding families: an anonymized LINCS L1000 perturbation screen (masked table/column/value names, with the held-out dictionary as ground truth) and TCGA colorectal tumor gene-expression matrices.
  • Backbone. Four agents on the same Gemini models (gemini-2.5-pro, retested on gemini-3.1-pro-preview) — so the scaffold, not the model, is the variable. K-Dense is the exception: its code executor is hardwired to Claude, and we ran it on Claude Opus 4.8 (its strongest coder), which makes it a cross-model-family check.
  • Ground truth is the hard part. We report only cases whose correct answer we re-derived and confirmed robust. An earlier draft of this page featured an "analysis-flag" case that, on re-verification, turned out to be driven by a sparse, outlier-heavy subset (one arm had two data points; one had none). We withdrew it. A later cross-check strengthened the call: the strongest coder we tested (Claude Opus 4.8, via K-Dense) independently judged the analysis-record flag unnecessary for that question — reaching the same all-records answer we now treat as correct. Getting the analysis population right is exactly the discipline this work is about — so we hold our own benchmark to it.
  • The 13 semantic-grounding cases were each re-verified from raw data. The masked-LINCS counts (3492, 622, 63367, 28195) and the join cases (fan-out 0.2494 vs 0.3007; 10726 dropped rows) are deterministic — recomputed directly against the value map and the join keys. The zero-as-missing cases are the only interpretation-dependent ones, so we checked them hardest: our own enricher's zero-handling detector scores these gene columns 0.88–0.94 "treat zeros as missing" — and, as a control, fires the opposite ("zeros are biological, keep them") on a lung transporter gene where excluding them would be the error. The missing-table case is verified by the table simply not being in the delivery. None are outlier-driven.
  • Scope. A controlled taxonomy of failure modes, not a leaderboard. The point is which families a stronger model closes (conventions) and which it never touches (grounding) — and therefore where a semantic layer, not a bigger model, is the fix.
